# Flowcharts

Use this reference for process, decision, approval, and hierarchical workflows.

## Shape semantics

- Start/end: terminal or rounded process shape.
- Action: process rectangle.
- Decision: diamond with short question text.
- Junction: small circle only when a merge or branch point is semantically useful.
- Note: borderless explanatory text outside the main path.

## Layout

- Choose top-to-bottom or left-to-right before placing nodes.
- Keep the main path on one center axis.
- Keep decision exits symmetric where possible.
- Use orthogonal connectors and concise branch labels.
- Move branch labels away from both the diamond border and connector.

## Quality checks

- consistent process-box family;
- decision text fits the diamond's smaller safe interior;
- outgoing branches have unambiguous direction;
- no backward edge without an intentional loop;
- no text-line, text-frame, or branch-label overlap.

Microsoft's basic flowchart templates are a useful semantic reference: [featured Visio templates and diagrams](https://support.microsoft.com/zh-TW/Visio/featured-visio-templates-and-diagrams).
