# Contributing

Contributions should follow the repository governance in `AGENTS.md` and the AiCoding maintenance workflow.

## Basic rules

- Keep canonical Skill source in the owning repository and do not edit generated plugin output by hand.
- Update README, README_CN, CHANGELOG, and related docs when behavior, installation, release, or workflow contracts change.
- Run the repository verification commands before proposing or merging changes.
- Use typed commit messages such as `feat(scope): summary`, `fix(scope): summary`, `docs(scope): summary`, or `build(scope): summary`.

## Pull requests

Include the reason for the change, validation commands and results, compatibility impact, documentation updates, and any manual gates that remain.
