# 固件生产与量产约束

## 1. 必问问题

```text
是否量产？
是否现场升级？
是否需要 OTA？升级通道是什么？
是否需要 A/B 或固定 App + 备份区？
是否需要 Manifest、CRC、Hash、签名、防降级？
是否需要回滚？回滚由 Bootloader 自动执行还是人工介入？
是否存在参数区、校准区、SN、MAC、License？
是否需要产线烧录记录、版本追溯、出厂测试报告？
是否需要锁调试口？锁死后如何售后恢复？
```

## 2. 量产固件最低要求

```text
固件版本号、构建 ID、Git commit、构建时间可读。
镜像校验独立于应用执行。
Bootloader 和 App 边界清晰。
升级失败有确定状态。
参数区和校准区不被 App 升级误擦除。
产线烧录和 OTA 使用同一 Manifest/版本模型。
```

## 3. 需要客户确认的升级策略

| 策略 | 优点 | 风险 | 适用 |
|---|---|---|---|
| A/B 双 App | 可靠回滚 | 链接地址和存储开销复杂 | 高可靠 OTA |
| 固定 App + 备份区搬运 | App 地址稳定 | 上电搬运/断电恢复复杂 | Bootloader 控制力强 |
| 单 App 原地升级 | 简单 | 断电风险高 | Demo/工厂线刷 |
| 外部主机托管升级 | 设备端简单 | 依赖主站 | EtherCAT/CAN/上位机场景 |
