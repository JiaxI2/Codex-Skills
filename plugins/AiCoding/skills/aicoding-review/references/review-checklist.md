# 嵌入式评审检查清单

当评审需要 checklist pass 时读取此参考。

## C99 与模块边界

- 公共头文件自包含。
- 头文件只包含声明，不定义变量或私有状态。
- 文件私有 helper 和状态使用 `static`。
- 接口校验空指针、范围、状态前置条件和失败路径。
- 不无故格式化或重构已稳定代码。
- 保持编码格式，尤其是 GBK 源文件或 Markdown 文件。

## 安全与实时性

- ISR 路径短小、有界、非阻塞。
- 实时路径无动态内存、递归、VLA、无界循环或不可控日志。
- 共享状态有明确同步策略。
- watchdog、timeout、安全状态和故障恢复行为清晰。

## 硬件与协议

- 寄存器访问处理 W1C、保留位、只读位和必要的清除/顺序语义。
- DMA/cache 所有权和 barrier 明确。
- 通信解析器在访问字段前校验长度、大小端、版本和 CRC/checksum。
- 外部输入默认不可信。

## 文档与可追溯性

- 项目约定要求时，新文件包含文件头。
- 新公共函数包含用途、参数、返回值和上下文约束。
- 关键 bug fix 记录现象、根因和修复方案。
- 项目要求审计性时，保留 Modification History。
